Как это работает

Посмотрите на ключевые функции бота и процессы, которые он автоматизирует. Каждый шаг продуман для максимальной эффективности и удобства.

Создание апелляции

Процесс, имитирующий реальный суд для максимальной объективности.

1
Пользователь подает доказательства (ссылки, файлы).
2
Бот задает 3 стратегических вопроса для сбора аргументов.
3
Запускается 24-часовой таймер для сбора контраргументов.
4
Дело автоматически передается ИИ для вынесения вердикта.

Вопрос по правилам

Получите точный ответ на любой вопрос по уставу за секунды.

1
Пользователь вводит команду /ask и свой вопрос.
2
Бот отправляет вопрос и файл rules.txt в API Google Gemini.
3
ИИ анализирует вопрос в контексте правил.
4
Пользователь получает развернутый и понятный ответ.

Создание поста (TextCrafter)

Простой пошаговый диалог для создания сложных публикаций.

1
Ответственный вводит команду /craft.
2
Бот последовательно запрашивает фото, текст, текст кнопки и ссылку.
3
"Машина состояний" запоминает прогресс.
4
Готовый пост публикуется по единому стандарту.

Часто задаваемые вопросы

Ответы на вопросы, которые могут у вас возникнуть по поводу безопасности, надежности и использования бота.

Зачем вообще нужен ИИ для апелляций?

Использование ИИ в качестве арбитра позволяет достичь максимальной объективности. В отличие от человека, ИИ не имеет личных симпатий, не устает и не подвержен эмоциям. Его решение основывается исключительно на фактах (аргументах сторон) и законе (уставе проекта). Это гарантирует, что каждое решение будет рассмотрено по единому, справедливому стандарту.

Насколько безопасно передавать данные боту?

Полностью безопасно. Бот размещен на надежной платформе Railway, данные хранятся в защищенной базе данных PostgreSQL. Все секретные ключи и токены хранятся в виде зашифрованных переменных окружения. Код бота полностью открыт, и вы можете лично убедиться в отсутствии скрытых функций.

Что произойдет, если бот перезапустится во время моей апелляции?

Ничего страшного. Благодаря базе данных PostgreSQL, бот сохраняет состояние каждой апелляции после каждого шага. Если он перезапустится, фоновый процесс автоматически найдет все "зависшие" дела и продолжит их обработку с того места, где они были прерваны. Ваши данные не будут потеряны.
